# «Мой заказ» — официантская система для суши-бистро (Telegram Mini App)

Работа официанта в духе **iiko**, но в виде Telegram Mini App:

| iiko | «Мой заказ» |
|------|-------------|
| Столы в зале | Реальные столы, которые можно переставлять в нужном порядке |
| Меню заведения | Общее меню суши-бистро (роллы, суши, сеты, горячее, напитки…) |
| Открытый счёт стола | Отдельный заказ под каждым столом, сохраняется до закрытия |
| Закрытие чека | Перенос заказа в историю, стол освобождается |

Чистый HTML + CSS + JavaScript (ES6+), без фреймворков.

> **О меню.** В приложение заложено реалистичное меню суши-бистро с московскими ценами в рублях как стартовый набор. Точного публичного меню заведения ровно с названием «Суши Бистро на Тверской» в открытом виде найти не удалось (у Тверской несколько суши-мест), поэтому меню сделано **полностью редактируемым прямо в приложении** — переименуйте, переоцените, добавьте или удалите любое блюдо под нужное заведение.

## Состав

```
index.html   — разметка всех экранов и модальных окон
styles.css   — оформление в духе iiko, цвета из темы Telegram
app.js       — логика: меню, столы, заказы, история, Telegram API
README.md    — этот файл
```

## Что умеет

- **Столы:** сетка плиток. Занятый стол подсвечивается и показывает сумму и число позиций. Кнопка «＋» добавляет стол, долгое нажатие (на ПК — правый клик) — редактирование (название, значок, цвет) и удаление.
- **Перестановка столов:** кнопка «↕ Расставить» переводит список в режим сортировки со стрелками ↑/↓. Нажмите «Готово», когда расставите.
- **Меню заведения:** разделы и карточки блюд с ценой. Тап — добавить в заказ, долгое нажатие — отредактировать или удалить блюдо. Кнопка «＋» добавляет новое блюдо (значок, название, цена, раздел или новый раздел).
- **Модификаторы блюда:** количество, чипы (Без васаби / Без имбиря / Острее / С собой), цена за порцию, комментарий повару.
- **Заказ стола (открытый счёт):** список позиций, свайп влево — удалить, тап — изменить, сумма заказа. Каждый стол держит свой заказ независимо.
- **Закрытие стола:** кнопка **MainButton** Telegram «Закрыть стол · оплачено» — заказ уходит в историю, стол освобождается.
- **История:** список закрытых заказов с датой, столом и суммой; просмотр позиций; «Повторить на этот стол».
- **Хранение:** `localStorage` + экспорт/импорт всей базы в JSON (⚙️). Структура готова к серверному сохранению.
- **Тема:** цвета подстраиваются под светлую/тёмную тему Telegram, поддержка безопасных зон.

## Локальный запуск (отладка)

Откройте `index.html` в браузере. Вне Telegram работает в режиме отладки: вместо нативной кнопки на экране заказа появляется обычная кнопка «Закрыть стол». Свайпы удобно проверять в эмуляции мобильного в DevTools.

## Публикация и подключение к боту

Telegram Mini App открывается **только по HTTPS**. Разместите файлы на статическом хостинге (GitHub Pages, Netlify, Vercel, Cloudflare Pages).

### GitHub Pages

1. Создайте публичный репозиторий, загрузите `index.html`, `styles.css`, `app.js` (файлы — в корень, не папкой).
2. **Settings → Pages → Source: Deploy from a branch**, ветка `main`, папка `/(root)`, **Save**.
3. Через 1–2 минуты появится адрес вида `https://ваш-логин.github.io/имя-репозитория/`.

### Создание бота в BotFather

1. В Telegram откройте **@BotFather** → `/newbot`.
2. Задайте имя и username (оканчивается на `bot`).
3. Сохраните токен (нужен только для будущего серверного сохранения).

### Привязка приложения

**Кнопка меню:** `/mybots` → ваш бот → **Bot Settings → Menu Button → Configure menu button** → пришлите URL приложения → задайте подпись.

**Отдельное Mini App со ссылкой:** `/newapp` → бот → название, описание, картинка 640×360, URL → короткое имя → получите ссылку `https://t.me/имя_бота/короткое_имя`.

### Обновление файлов

После перезаливки файлов Telegram может показывать старую версию из кэша. Очистите кэш: **Настройки Telegram → Данные и память → Очистить кэш**.

## Готовность к серверному хранению

Всё состояние — один объект `DB.state`:

```json
{
  "version": 2,
  "settings": { "currency": "₽" },
  "menu":   { "sections": [ { "id":"...","name":"Роллы","items":[ {"id":"...","emoji":"🍣","name":"Филадельфия","price":520} ] } ] },
  "tables": [ { "id":"...","name":"Стол 1","emoji":"1️⃣","color":"c2","order":[ /* строки заказа */ ] } ],
  "history": [ /* закрытые заказы */ ]
}
```

Чтобы хранить на сервере: проверяйте `Telegram.WebApp.initData` на бэкенде (по токену бота), отправляйте `DB.state` в `DB.save()` и загружайте его при старте вместо `localStorage`. Менять разметку не придётся.

## Подсказки

- **Тап по столу** — открыть его заказ (если пуст, сразу меню).
- **Долгое нажатие по столу** — изменить/удалить; **«↕ Расставить»** — режим перестановки.
- **Долгое нажатие по блюду** — изменить цену/название или удалить блюдо.
- **Свайп влево по позиции** в заказе — удалить; **тап** — изменить модификаторы.
- **⚙️** — экспорт/импорт JSON и сброс к стандартному меню.
